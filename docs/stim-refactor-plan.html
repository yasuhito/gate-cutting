Stim リファクタリング作業計画

このメモは「Stim の箇所をリファクタリングして使えるようにする」ための具体的な作業分解です。

結論: まず作るもの

まずは重複している Stim 関連処理を、実験スクリプトから切り出して共通モジュールにします。

推奨構成:

src/gate_cutting/
├── __init__.py
├── device.py          # device.json -> ErrorParams / fidelity / 座標
├── stim_backend.py    # Qiskit -> Stim, ノイズ挿入, 実行, 期待値計算
└── gate_cutting.py    # Gate Cutting の分解・切断実行

tests/
├── test_device.py
├── test_stim_backend.py
└── test_gate_cutting.py

研究コードをすぐ使える形にするだけなら、最初は src/ なしで gate_cutting/ 直下に置いてもよいです。ただし将来パッケージ化するなら src/gate_cutting/ が無難です。

1. 共通化する対象

現状、以下に同じような処理が重複しています。

  • legacy/ex2/ev1.py
    • load_device_data()
    • qiskit_to_stim()
    • append_operation_with_noise()
    • simulate_ideal() / simulate_noisy() / simulate_cut_circuit()
  • experiments/exp1/b1.py
    • ErrorParams
    • qiskit_to_stim()
    • append_operation_with_noise()
    • GateCutStimSimulator
  • experiments/exp2/b1.py, experiments/exp2/b2.py
    • DeviceManager

最初のリファクタリングでは experiments/exp2/b2.py をベースにしつつ、legacy/ex2/ev1.py の load_device_data() と wire-cut 関連の知見を拾うのがよいです。

2. 最低限ほしい公開API

device.py

@dataclass(frozen=True)
class ErrorParams:
    one_qubit: dict[int, float]
    two_qubit: dict[tuple[int, int], float]
    readout: dict[int, float]

@dataclass(frozen=True)
class DeviceData:
    raw: dict
    cx_fidelities: dict[tuple[int, int], float]
    one_q_fidelities: dict[int, float]
    qubit_coords: dict[int, tuple[float, float]]
    error_params: ErrorParams

def load_device(path: str | Path) -> DeviceData: ...
def parse_device(data: dict) -> DeviceData: ...

ポイント:

  • device.json から ErrorParams を取る処理を1箇所に集約する。
  • reverse_fidelity がある場合は逆方向も入れる。
  • 2Q error は 1 - fidelity として持つ。

stim_backend.py

def qiskit_to_stim(qc: QuantumCircuit, *, insert_ticks: bool = True) -> stim.Circuit: ...
def active_qubits(circuit: stim.Circuit) -> list[int]: ...
def append_operation_with_noise(circuit: stim.Circuit, op_name: str, targets: list[int], error_params: ErrorParams) -> None: ...
def add_noise(circuit: stim.Circuit, error_params: ErrorParams) -> stim.Circuit: ...
def ensure_measurements(circuit: stim.Circuit, error_params: ErrorParams | None = None) -> stim.Circuit: ...
def parity_expectation(samples: np.ndarray) -> float: ...
def sample_expectation(circuit: stim.Circuit, shots: int) -> float: ...
def run_standard(circuit: stim.Circuit, shots: int, error_params: ErrorParams | None = None) -> float: ...

ポイント:

  • Qiskit 変換、ノイズ挿入、測定追加、期待値計算を分離する。
  • 実験コードから直接 Stim の細かい処理を触らなくて済むようにする。

gate_cutting.py

@dataclass(frozen=True)
class CutTarget:
    instruction_index: int
    qubits: tuple[int, int]

CX_DECOMPOSITION = [
    (0.5, "I", "I"),
    (0.5, "Z", "I"),
    (0.5, "I", "X"),
    (-0.5, "Z", "X"),
]

def find_cx_cut_targets(circuit: stim.Circuit, cut_pairs: list[tuple[int, int]]) -> list[CutTarget]: ...
def run_gate_cut(circuit: stim.Circuit, cuts: list[CutTarget], error_params: ErrorParams, shots: int) -> float: ...

ポイント:

  • 今の run_cut() は (u, v) のペアで切断対象を探すため、同じペアの CX が複数あると全部切ってしまう可能性があります。
  • 可能なら ペアだけでなく instruction index / gate id で切る 形に直すべきです。

3. 重要な修正ポイント

3.1 CX 後の I を TICK 等に置き換える

現状:

stim_circ.append("CX", qs)
stim_circ.append("I", qs)

これは「Stim が同じ命令をまとめると、後で CX ごとにノイズを入れにくい」ことへの暫定対応です。

やること:

  • I ではなく TICK を入れる案をまず試す。
  • TICK は Stim の annotation なので、「ここで区切る」という意図が I より明確。
  • append_operation_with_noise() では TICK はノイズを付けずにそのまま通す。

候補:

elif op in ("cx", "cnot"):
    stim_circ.append("CX", qs)
    if insert_ticks:
        stim_circ.append("TICK")

注意:

  • TICK を入れた場合に、既存ベンチマークの結果が大きく変わらないことを確認する。
  • Qiskit の barrier を Stim の TICK に変換するのもあり。

3.2 自動測定時にも readout error を入れる

現状の一部コードでは、測定がない場合に最後へ M を追加していますが、その自動追加測定に readout error が入らない実装があります。

やること:

  • ensure_measurements(circuit, error_params) に集約する。
  • error_params がある場合は、測定前に X_ERROR(p) を追加してから M する。

3.3 2Q error の向きの扱いを統一する

現状、コードによって (u, v) だけを見るものと、なければ (v, u) も見るものがあります。

やること:

  • lookup_two_qubit_error(error_params, u, v) のような関数を作る。
  • デバイスが方向付き Fidelity を持つなら方向を尊重する。
  • 方向なしとして扱いたい場合だけ fallback で逆向きを見る。

3.4 unsupported gate は黙って無視しない

現状の qiskit_to_stim() は、対応していないゲートを暗黙に無視する可能性があります。

やること:

  • 未対応ゲートは基本 NotImplementedError を出す。
  • strict=False のときだけ warning して skip する。

例:

else:
    raise NotImplementedError(f"Unsupported Qiskit gate for Stim conversion: {op}")

3.5 切断対象は (u, v) だけでなく gate index で持つ

現状:

cut_pairs = [(15, 14)]

問題:

  • 同じ (15, 14) の CX が回路中に複数あると、すべて切断される可能性がある。
  • MIP は MultiDiGraph を使っているのに、最終的に (u, v) へ潰している箇所がある。

やること:

  • MIP 側の出力を CutTarget(instruction_index, qubits, fidelity) のようにする。
  • 少なくとも Stim 側では cut_indices を明示的に渡せるようにする。

4. テストで最低限確認すること

tests/ を作り、以下を自動テスト化すると安心です。

  1. device parse

    • experiments/exp2/device.json を読める。
    • qubits 数、couplings 数、worst fidelity が期待通り。
    • ErrorParams.two_qubit[(15, 14)] == 1 - 0.5234 のような確認。

  2. Qiskit -> Stim 変換

    • H(0); CX(0,1); measure_all() が H, CX, M を含む Stim 回路になる。
    • CX 後に TICK が入る。

  3. ノイズ挿入

    • X 0 後に DEPOLARIZE1(p) が入る。
    • CX 0 1 後に DEPOLARIZE2(p) が入る。
    • 測定時に readout 用 X_ERROR(p) が入る。

  4. 自動測定

    • 測定なし回路を run_standard(..., error_params=...) したとき、最後の測定にも readout error が入る。

  5. Gate Cutting の sanity check

    • ノイズなし・十分な shots で、切断なしと切断ありの期待値が大きくずれない。
    • 同じ qubit pair の CX が複数ある場合、指定した index だけ切れる。

5. 作業順序

Step 1: 共通モジュールを作る

  • src/gate_cutting/device.py
  • src/gate_cutting/stim_backend.py
  • src/gate_cutting/gate_cutting.py

まずは既存コードのコピペでもよいので、APIを固定する。

Step 2: experiments/exp2/b2.py を共通モジュール利用に置き換える

b2.py が一番まとまっているので、最初の移行対象にする。

Step 3: experiments/exp2/benchmark_phase1.py / benchmark_phase2.py を確認

ベンチマークが従来通り走ることを確認する。

Step 4: experiments/exp1/b1.py も置き換える

同じ qiskit_to_stim() / ノイズ挿入を使うようにする。

Step 5: legacy は触りすぎない

legacy/ex2/ は参考として残す。無理に全部直すより、README と docs で「参考コード」と明記する。

6. 最初のPR/コミットでやるとよい最小範囲

最初から全部やると大きいので、最初はこれだけでよいです。

  1. gate_cutting/stim_backend.py を作る。
  2. ErrorParams, qiskit_to_stim, append_operation_with_noise, add_noise, run_standard を移す。
  3. CX 後の I を TICK にする。
  4. 自動測定時の readout error を直す。
  5. experiments/exp2/b2.py だけ新モジュールを使う。
  6. 小さいテストを3〜5個追加する。

これで「Stim の箇所を再利用可能にした」と言える状態になります。
