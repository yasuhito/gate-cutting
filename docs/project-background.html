Project Background

このページは、README から移動した背景情報です。論文付属の研究スクリプトを OQTOPUS 向けコンポーネントへ育てるため、元コードの位置づけ、保存方針、実験対応関係をまとめます。

プロジェクトの出発点

元の目的

量子クラウド環境を想定した Gate Cutting / Wire Cutting と、デバイス Fidelity 情報を使った MIP による切断箇所最適化 の実験コードを整理すること。

現在の目的

研究用に適当に作られたスクリプトを、OQTOPUS の一部として利用できる品質までリファクタリングし、テスト済みコンポーネントへ分解すること。

実験コードの対応関係

場所位置づけ主な内容
experiments/exp1/ 原稿の第1段階実験 Gate Cutting と Wire Cutting の基礎比較。サンプリングオーバーヘッド、復元誤差、ショット数依存性を見る。
experiments/exp1/benchmark_phase1.py Qiskit Aer benchmark Gate Cutting / Wire Cutting の比較。切断数に対する復元誤差とサンプリングオーバーヘッドを評価。
experiments/exp1/b1.py Stim benchmark Stim を使った Gate Cutting のノイズスイープ実験。現在は共有 Stim / Gate Cutting helper を利用。
experiments/exp2/ 原稿の第2段階実験 実機想定の Fidelity / error 情報、MIP、Stim Gate Cutting を組み合わせる評価。
experiments/exp2/device.json device metadata 量子ビット Fidelity、2量子ビットゲート Fidelity、readout error、座標を含むデバイスデータ。
experiments/exp2/b1.py 統合版 MIP で低 Fidelity な CX ゲートを選び、Stim で Gate Cutting の効果を評価する。
experiments/exp2/b2.py 改良版 b1.py の改良版。max_cuts を守る処理やランダムデバイス生成機能を含む。
experiments/exp2/benchmark_phase1.py fixed-device benchmark 固定 device.json を使った、最大切断数ごとのエラー比較。
experiments/exp2/benchmark_phase2.py random-device benchmark 試行ごとにランダムデバイスを生成するベンチマーク。
experiments/exp2/check.py visual check デバイスグラフ上で使用ゲート・切断ゲートを可視化する確認用スクリプト。

作者コメント・旧実装から拾った知見

Stim 変換

  • 旧実装には qiskit_to_stim() が複数存在した。
  • CX 後に I を入れる暫定対応があった。
  • 現在は意図が明確な TICK に置き換え済み。

ノイズ挿入

  • device.json から Fidelity / readout error を取得する。
  • ErrorParams を使って DEPOLARIZE1, DEPOLARIZE2, readout X_ERROR を挿入する。
  • 現在は gate_cutting.stim_backend と gate_cutting.device に集約済み。

legacy の扱い

legacy/ex2/ は試作・参考コードです。原稿本体の実験再現は experiments/exp1/ と experiments/exp2/ を優先します。

方針: legacy は必要以上に書き換えない。新しい品質保証や OQTOPUS 統合に向けた実装は gate_cutting/ と tests/ に集約する。

参考になる旧実装

  • legacy/ex2/ev1.py — load_device_data(), qiskit_to_stim(), append_operation_with_noise(), cut simulation の参考。
  • legacy/ex2/e4.py, legacy/ex2/e5.py — MIP と実験過程の比較資料。

整理メモ

  • 元の qCxT.zip は archive/original/qCxT.zip に保管。
  • qCxT (1).pdf は qCxT.pdf と同一内容だったため、重複PDFとして archive/original/qCxT-duplicate.pdf に移動。
  • zip 内の macOS メタデータ（.DS_Store, __MACOSX/）は展開対象から除外。
  • experiments/exp2/benchmark_phase1.py / benchmark_phase2.py は、同じディレクトリ前提の from b2 import ... ではなく、プロジェクトルートを sys.path に追加して from experiments.exp2.b2 import ... する形に整理済み。
