src/gate_cutting

実装メモ

src/gate_cutting/ が提供する処理の流れ、主要モジュール、入出力データ、実行時の前提を開発者向けにまとめます。

処理の全体像

1. 回路入力Qiskit 回路または互換オブジェクト
2. デバイス読込フィデリティ・読み出し誤差・座標
3. CX 候補化CircuitEdge と命令番号
4. MIP 選択SciPy MILP で CutTarget を選択
5. Stim 実行標準実行またはノイズ付き実行
6. Gate Cuttingサブ回路期待値の重み付き合成

入出力

入力

Qiskit 回路 device.json shots max_cuts cut_fidelity_threshold

→

出力

標準実行の期待値 Gate Cutting 後の期待値 CutTarget 一覧 フィデリティ付き切断情報

主要データ構造

型責務主なフィールド
ErrorParams Stim に挿入するエラー確率を保持する。 one_qubit, two_qubit, readout
DeviceData デバイス JSON から抽出したフィデリティ、座標、エラー情報をまとめる。 cx_fidelities, one_q_fidelities, qubit_coords, error_params
CircuitEdge MIP の辺変数に対応する CX 候補を表す。 edge_index, instruction_index, qubits, fidelity
CutTarget 切断する具体的な Stim の CX 命令を表す。 instruction_index, qubits, fidelity

Stim 変換とノイズ挿入

変換規則

x → X y → Y z → Z h → H s → S sdg → S_DAG cx → CX + TICK measure → M barrier → TICK

ノイズ

1 量子ビットゲート後: DEPOLARIZE1 CX 後: DEPOLARIZE2 測定前: X_ERROR TICK にはノイズを付けない

MIP と CutTarget の対応

Qiskit 命令列CX を走査
CircuitEdge辺番号とフィデリティ
MIP 変数辺ごとの切断判定
選択された辺選ばれた辺番号
CutTargetStim 命令番号
run_gate_cut該当 CX だけ置換

同じ (control, target) の CX が複数ある場合も、instruction_index により切断対象を一意に指定します。

実行時の扱い

未対応ゲート

qiskit_to_stim(..., strict=True) では NotImplementedError を送出します。strict=False では警告を出してスキップします。

測定

測定が無い Stim 回路は、実行前に使用中の量子ビットへ測定を追加します。ノイズ付き実行では読み出し誤差も追加します。

MIP ソルバー

SciPy の milp を使います。ソルバーが失敗した場合は例外として扱います。
