Stim 変換のポイント

旧: Identity で区切る

stim_circ.append("CX", qs)
stim_circ.append("I", qs)

命令の結合を避けたい意図はあるが、回路へ identity gate を混ぜてしまう。

→

新: TICK で区切る

stim_circ.append("CX", qs)
stim_circ.append("TICK")

TICK は annotation として区切り意図が明確で、ノイズ挿入位置も追跡しやすい。

x → X y → Y z → Z h → H s → S sdg → S_DAG cx/cnot → CX + TICK measure → M

デバイスデータからノイズ付き Stim 回路へ

device.json から読むもの

量子ビット ID 座標 1Q Fidelity readout error CX Fidelity

挿入する代表的ノイズ

1Q 後: DEPOLARIZE1 CX 後: DEPOLARIZE2 測定前: X_ERROR

Gate Cutting 展開

CX を切る = 0.5 × ( II + ZI + IX − ZX )

CutTarget は instruction_index と qubits を持ちます。同じ (control, target) の CX が複数あっても、MIP が選んだ具体的な CX だけを切れます。

MIP は SciPy 必須

研究実行環境

NetworkX / SciPy を必須依存として入れ、CX をグラフ edge として扱い、SciPy の milp で切断対象を選びます。

fallback なし

greedy fallback は廃止しました。SciPy が無い、または MILP solver が失敗した場合は、結果を曖昧にせずエラーとして止めます。
